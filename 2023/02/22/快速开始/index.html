快速开始

发表于 2023-02-22 更新于 2023-04-14 分类于 帮助

本章的主要内容是：指导大家如何快速的创建并发布一篇文章。

搭建简介

该博客系统使用hexo框架+nextT主题搭建的，部署在阿里云服务器上自建的GitLab上。

hexo搭建的博客页面是一个静态的页面，不支持在线编辑和上传文章。所以如果要发布一篇文章的话，需要把博客源码下载到自己的本地笔记本上，通过hexo命令进行文章的创建和发布。所以借助GitLab来管理大家的对博客源码的提交。

博客源码推送到GitLab远程仓库的dev分支后，通过gitlab-ci.yml中配置的脚本自动执行编译，编译后在仓库目录下生成public文件，public文件中存放的就是博客的静态页面。最后通过nginx配置监听4000端口来访问public目录下的index.html页面。

NOTE : 为何不用GitLab-Pages?
原因是 : GitLab-Pages 的前提是需要有个域名并配置一个通配符DNS 记录.

准备工作

在开始创作文章之前，需要先获取博客源码并且在本地电脑上搭建开发环境。

  1. 博客是通过hexo框架搭建的，所以在创作文章前需要先在本地电脑安装hexo。安装文档连接： https://hexo.io/zh-cn/docs/。我们强烈建议永远安装最新版本的 hexo，以及 推荐的 Node.js 版本。

  2. 博客源码是托管在自建的GitLab仓库中的，所以如果要新增内容就需要先注册git账号。注册好之后通知管理员开通仓库访问权限。

  3. 进入仓库后，把博客源码clone到自己本地电脑里，切换到dev分支。因为博客的源码是在dev分支，main分支是源码编译后的静态页面。

  4. 源码下载到本地后，cd 到源码根录下，执行hexo s 命令就可以在本地进行博客服务的启动，网文地址为：http://localhost:4000/。

文章创作

准备工作完成后，就可以进行文章的创作。文章的创作涉及文章新建、删除、预览、资源管理、markdown语法等问题，这里以名为演示的这篇文章的创建给大家举例。

Tips: 建议大家使用vscode编辑器来编写文章比较方便

  1. 在博客源代码dev分支的根目录下输入命令：hexo new 快速开始 来快速新建文章。该命令会在项目的source/_posts目录下创建演示.md文件和同名的文件夹，演示.md文件就是用来编写文章的，同名文件夹是用来存放文章里面要用到的图片资源。

  2. 在演示.md中就可以进行文章的编写了，文章编写要遵循以下几点规范：

  • 文章要标注作者、日期、归类、标签
  • 文章要有首页图片和概要介绍，图片的位置和概要介绍放置在<!-- more --> 标签之前
  • 文章的基本层次结构为： 概要、内容、总结
  • 文章不要出现一级标题#, 要从二级标题开始##
  • 样例图片如下：

  1. 文章在创作过程中可以通过命令hexo s 来预览，以方便及时调整排版。

  2. 在创作文章时图片的插入是经常会使用的, 图片要保存在文章同名文件夹下，在文章中插入时可以用![hexo-new.jpg](hexo-new.jpg)的方式来实现(hexo-new是我的图片名称)。

  3. 文章若要删除，或者重命名时，需要把同名的文件夹也一并删除或重命名。

  4. 项目的.gitlab-ci.yml文件不要随便修改。

文章发布

文章创作完成后，在本地先通过hexo s进行预览。确认没有问题后就可以把代码提交push到GitLab的远程仓库了。推送成功后稍等片刻，等待CI自动编译完成，就可以通过网址：http://47.95.10.124:4000/ 来访问自己新发布的文章了。

总结

博客文章的创建和发布还是比较方便的，开发人员只需要把它当作普通的git项目进行开发维护就可以。所以编写博客的前提条件是要注册一个git账户并赋予仓库的访问权限。比较麻烦的是开发人员需要在本地搭建nodejs环境，这对于后端开发的同事来说是比较陌生的领域，不过好在这些都不是太复杂。
